dbRIP — Database of Retrotransposon Insertion Polymorphisms

Track Description

This track displays polymorphic mobile element insertions (MEIs) from the dbRIP database maintained by the Liang Lab at Brock University. dbRIP catalogs transposable element insertion polymorphisms across human populations, derived from the 1000 Genomes Project. Each item represents a single insertion locus; clicking an item shows its dbRIP ID which can be used to query the full dataset via the dbRIP API.

The database currently contains 44,984 TE insertion polymorphisms genotyped across 33 populations (2,504 individuals).

Display Conventions and Configuration

Insertions are grouped into sub-tracks by mobile element family. Each sub-track can be independently shown, hidden, or reconfigured via the track controls panel.

Sub-track Color Insertions Description
dbRIP ALU Blue 33,709 Polymorphic Alu SINE insertions
dbRIP LINE1 Vermilion 6,468 Polymorphic LINE-1 (L1) insertions
dbRIP SVA Bluish green 4,697 Polymorphic SVA (SINE-R/VNTR/Alu) insertions
dbRIP HERVK Reddish purple 101 Polymorphic HERV-K endogenous retrovirus insertions
dbRIP PP Orange 9 Polymorphic processed pseudogene insertions
Future types Gray — Any additional ME families added in future dataset releases

Colors follow the Okabe-Ito colorblind-safe palette, which is distinguishable under deuteranopia, protanopia, and tritanopia.

Insertion IDs (e.g. A0000001) can be entered into the UCSC browser's search box to jump directly to a specific insertion. Type the ID and press Enter — the browser will navigate to that locus with the insertion highlighted.

Methods

Insertion coordinates were originally derived from the 1000 Genomes Project whole-genome sequencing data and catalogued in dbRIP. The hg38 coordinates in this track are sourced directly from the dbRIP dataset (assembly: GRCh38). Population allele frequencies represent the proportion of individuals in each 1000 Genomes population carrying the insertion.

Track files (bigBed format) are rebuilt automatically when the underlying dataset is updated, via a GitHub Actions CI pipeline that exports BED data from the dbRIP API, sorts and indexes it with UCSC's bedToBigBed tool, and deploys the result to GitHub Pages.
